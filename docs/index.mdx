# si Documentation

`si` provides one CLI surface for:

- Agent operations (`dyad`, codex container lifecycle)
- Secure secrets (`vault`)
- Cloud/provider bridges
- Docker-native PaaS operations (`si paas ...`)
- Browser MCP runtime (`si browser ...`)
- Plugin marketplace (`si plugins ...`)
- Docs workflow (`si mintlify ...`)

## Start here

1. Build runtime image: `si build image`
2. Validate credentials/secrets: `si vault status`
3. Check command map: `si --help`
4. Review operational runbooks before production deploys.
